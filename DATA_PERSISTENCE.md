# 驾校约课系统 - 数据持久化说明

## 系统架构

本系统采用 **数据库优先 + Mock 数据降级** 的架构设计：

```
┌─────────────────┐
│   数据库操作     │  ← 优先使用
└────────┬────────┘
         │ 失败时
         ↓
┌─────────────────┐
│   Mock 数据      │  ← 降级方案
└─────────────────┘
```

## 主要修改

### 1. 创建数据初始化模块 (`src/services/dataInitializer.js`)

**功能：**
- 测试数据库连接
- 从数据库加载用户、课程、预约数据
- 当数据库为空时自动初始化默认数据（admin、instructor1、instructor2、student1）
- 所有默认账号密码都是：`123456`

### 2. 更新应用启动 (`src/app.js`)

**功能：**
- 启动时先连接并初始化数据库
- 数据库连接成功后才启动服务
- 失败时仍能正常启动（使用 Mock 数据）

### 3. 更新用户数据模块 (`src/data/mockUserData.js`)

**功能：**
- 启动时从数据库加载用户数据
- 如果数据库为空，使用默认数据
- 用户操作同时更新数据库和 Mock 数据

### 4. 更新课程服务 (`src/services/courseService.js`)

**功能：**
- 启动时从数据库加载课程数据
- 统一字段映射（`instructor_id` ↔ `coach_id`）
- 课程操作同时更新数据库和 Mock 数据

### 5. 更新预约服务 (`src/services/appointmentService.js`)

**功能：**
- 启动时从数据库加载预约数据
- 预约操作同时更新数据库和 Mock 数据
- 签到、标记缺席功能完整实现

## 使用步骤

### 1. 确保数据库配置正确

检查 `src/config/database.js` 中的数据库配置：

```javascript
host: '192.168.2.135',  // 数据库主机
port: 3306,
user: 'root',
password: '123456',
database: 'driving_school'
```

如果您的数据库配置不同，请修改这些值。

### 2. 初始化数据库

在 MySQL 中执行 `init.sql` 脚本：

```bash
mysql -u root -p < backend/init.sql
```

或者在 MySQL 管理工具（如 Navicat、phpMyAdmin）中导入该文件。

### 3. 启动后端服务

```bash
cd backend
npm install
npm start
```

启动时会看到：
```
正在连接数据库...
数据库连接成功！
从数据库加载了 X 个用户数据
从数据库加载了 X 个课程数据
从数据库加载了 X 个预约数据
驾校约课系统后端服务运行在 http://localhost:3002
```

### 4. 启动前端服务

```bash
cd frontend
npm install
npm run dev
```

## 数据持久化说明

### ✅ 现在可以正常工作的功能：

1. **用户注册**
   - 新注册的用户会写入数据库
   - 重启服务后用户数据依然存在
   - 可以用注册的用户名和密码正常登录

2. **课程管理**
   - 添加的课程会写入数据库
   - 编辑的课程会更新数据库
   - 删除的课程会从数据库删除
   - 重启服务后课程数据依然存在

3. **预约功能**
   - 学员预约课程会写入数据库
   - 取消预约会更新数据库
   - 教练签到会更新数据库
   - 教练标记缺席会更新数据库
   - 重启服务后预约数据依然存在

4. **用户管理**
   - 添加用户、删除用户、启用/禁用用户、重置密码都会同步到数据库

### 📊 数据同步机制

所有数据操作都遵循：
1. **优先尝试数据库操作**
2. **如果成功，同时更新 Mock 数据**（保持内存状态一致）
3. **如果失败，仅使用 Mock 数据**（保证系统可用性）

## 故障排查

### 问题1：数据库连接失败

**现象：** 启动时看到 "数据库连接失败" 的错误信息

**解决方案：**
1. 检查数据库服务是否启动
2. 检查 `database.js` 中的配置是否正确
3. 确认数据库 `driving_school` 是否已创建
4. 执行 `init.sql` 初始化表结构

### 问题2：数据不保存

**现象：** 重启服务后数据丢失

**解决方案：**
1. 确认数据库连接正常（看启动日志）
2. 检查数据库中是否有对应表
3. 查看操作是否有错误日志

### 问题3：仍然使用 Mock 数据

**现象：** 操作成功但数据库没有数据

**原因：** 数据库操作失败，系统自动降级到 Mock 数据

**解决方案：**
1. 检查数据库连接和权限
2. 查看后端错误日志
3. 修复数据库问题后重启服务

## 注意事项

1. **数据库必须先初始化**：首次使用前必须执行 `init.sql`
2. **字段名称**：注意 `instructor_id` 和 `coach_id` 的映射关系
3. **密码加密**：所有密码都使用 bcrypt 加密存储
4. **ID 自增**：数据库使用自增 ID，Mock 数据也会同步
